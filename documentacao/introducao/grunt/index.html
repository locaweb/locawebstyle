Grunt Tasks

Entenda quais as tasks você pode usar no projeto

Grunt é um task runner desenvolvido para automatizar algumas tarefas corriqueiras durante o desenvolvimento web. Aqui no Locaweb Style, temos algumas tarefas que você pode usar durante seu desenvolvimento e abaixo listamos todas elas.

Rodando a primeira vez

O Grunt tem uma coleção de tasks que tem algumas dependencias e precisam ser instaladas antes de rodarem. Para isso, apenas digite no console: npm install.

Feito isso, você pode rodar as tasks abaixo.

grunt

Este é o comando default. Rodando grunt no console ele irá rodar algumas tarefas básicas descritas no Gruntfile.js. Por enquanto definimos como padrão as tarefas: jshint e plato. Mas a ideia é ter posteriormente rodando outras tasks úteis.

grunt test

Esse comando roda todas as tasks que testam nosso código. Ele vai rodar por exemplo JSHint e Jasmine. Se você quiser testar seu código basta rodar essa tarefa

grunt jshint

JSHint é uma ferramenta que te ajuda a escrever JavaScripts com o mínimo de erros de sintaxe que podem causar problemas em potencial futuramente no projeto. O plugin do NPM que usamos é o grunt-contrib-jshint.

Para rodar o JSHint no projeto, apenas digite no console: grunt jshint.

grunt jasmine

O Jasmine vai testar seus scripts. Se vocÊ rodar apenas no conesole um grunt jasmine, ele vai rodar todos os testes do Locaweb Style que se encontram na pasta /spec/ do projeto. Se você rodar um grunt test, o Grunt rodará os todas as tarefas relacionadas a testes, no caso, por enquanto, só o JSHint e o Jasmine.

grunt githooks

O Githooks roda uma série de comandos relacionados com o Git. Uma delas é o pre-commit, que roda o JSHint antes dos seus commits. Se você quiser ativar essas configurações, basta digitar no console grunt githooks.

Os hooks do Git podem ser vistos nesse site. Ainda não implementamos todos porque não há necessidade. Por enquanto há apenas o pré-commit que analisa seu código antes dos commits serem enviados.

grunt plato

O Plato gera um relatório (como esse) de complexidade de código e manutenção do JavaScript do projeto. Geralmente nós geramos no projeto de tempos em tempos e você não precisa se preocupar com isso. Para visualizar os resports, basta ver index.html que está dentro da pasta spec/report-js.
